## Step-by-Step Explanation

### Phase 1: Image Upload (Frontend)
1. **User Action**: User selects or drags an image file
2. **File Handling**: Browser reads file into memory
3. **FormData Creation**: JavaScript creates FormData with:
   - `image_file`: The image file
   - `limit`: Max results (default: 10)
   - `score_threshold`: Min similarity (optional)

### Phase 2: API Request (Network)
4. **HTTP POST**: Browser sends multipart/form-data request
5. **Server Receives**: FastAPI endpoint receives request

### Phase 3: Image Processing (Backend)
6. **File Storage**: Server saves uploaded file temporarily
7. **Vision Analysis**: GPT-4o analyzes image and generates description
8. **Embedding Generation**: Description converted to 1536-dim vector
9. **Vector Search**: Qdrant searches for similar vectors

### Phase 4: Results (Backend â†’ Frontend)
10. **Similarity Matching**: Qdrant returns top N similar images
11. **Metadata Retrieval**: Each result includes URI, score, and metadata
12. **Response Formatting**: Server formats JSON response
13. **Cleanup**: Temporary file deleted

### Phase 5: Display (Frontend)
14. **Response Parsing**: JavaScript parses JSON response
15. **UI Update**: Results displayed in grid with scores

## Data Flow Example

### Input
```javascript
// User uploads: duck_image.jpg
FormData {
  image_file: File { name: "duck_image.jpg", size: 245KB },
  limit: "10",
  score_threshold: "0.7"
}
```

### Processing
```python
# 1. Image analyzed by GPT-4o Vision
description = "A yellow duckling with orange feet swimming in a blue pond"

# 2. Description embedded
embedding = [0.123, -0.456, 0.789, ...]  # 1536 dimensions

# 3. Qdrant searches for similar vectors
results = qdrant.search(embedding, limit=10, threshold=0.7)
```

## Component Responsibilities

| Component | Responsibility |
|-----------|---------------|
| **Frontend** | File upload, UI display, API calls |
| **FastAPI** | Request handling, file management, response formatting |
| **ImageEmbeddingService** | Image analysis, embedding generation |
| **OpenAI Vision** | Image description generation |
| **OpenAI Embeddings** | Text-to-vector conversion |
| **Qdrant** | Vector storage and similarity search |
| **Knowledge Graph** | Metadata storage (optional) |

## Performance Characteristics

- **Image Upload**: ~100-500ms (depends on file size)
- **Vision Analysis**: ~1-3 seconds (GPT-4o API call)
- **Embedding Generation**: ~200-500ms (OpenAI API)
- **Qdrant Search**: ~10-50ms (vector similarity)
- **Total**: ~2-4 seconds end-to-end

## Error Handling

Each step includes error handling:

1. **Qdrant Down**: Returns 503 with helpful message
2. **Invalid File**: Returns 400 with validation error
3. **API Timeout**: Returns 500 with timeout error
4. **No Results**: Returns empty results array (not an error)

## Security Considerations

- File type validation (images only)
- File size limits
- Temporary file cleanup
- CORS configuration
- API authentication (if configured)
